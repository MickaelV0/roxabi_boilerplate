2. **Source data** using the hybrid approach: start with GitHub API / web, clone locally if the repo is complex

4. **Link back** to the GitHub sub-issue in the Context section
5. **All 16 analyses** will later be consolidated into a single summary document

## Analysis Axes (13 Total)

Each repo brainstorm must cover **all applicable axes** in the following order:

### 1. What It Does `{rating}`

- Purpose and value proposition
- Target audience
- Problem it solves

### 2. How It Works `{rating}`

- Core approach / methodology
- Key workflows and user interactions
- Runtime behavior

### 3. Architecture &amp; Layers `{rating}`

- High-level architecture (monolith, monorepo, microservices, etc.)
- Component boundaries and layers (UI, API, data, infra)
- Design patterns in use (DDD, hexagonal, event-driven, etc.)
- Dependency graph between modules

### 4. File / Project Structure `{rating}`

- Directory layout and naming conventions
- Monorepo organization (if applicable)
- Package/module boundaries
- Configuration file placement

### 5. Tech Stack &amp; Tooling `{rating}`

- Runtime (Node, Bun, Deno, etc.)
- Framework(s) (React, Vue, NestJS, etc.)
- Language &amp; type system
- Linter, formatter, bundler
- Package manager

### 6. Developer Experience (DX) `{rating}`

- Dev scripts and commands
- Dev server and hot reload
- Environment management (.env, secrets)
- Onboarding friction (time to first run)
- Editor/IDE integration

### 7. Testing Strategy `{rating}`

- Test types present (unit, integration, e2e)
- Test framework and runner
- Coverage targets and enforcement
- Test organization and naming

### 8. CI/CD Pipelines `{rating}`

- Pipeline platform (GitHub Actions, GitLab CI, etc.)
- Stages and checks (lint, type, test, build, deploy)
- Branch protection and merge rules
- Deployment strategy (preview, staging, production)
- Secrets management in CI

### 9. Documentation Quality `{rating}`

- README quality and completeness
- Architecture docs
- API docs / code comments
- Contributing guide
- Changelog and versioning

### 10. Unique / Novel Ideas `{rating}`

- Anything clever, creative, or unconventional
- Patterns we haven't seen before
- Tools or integrations worth exploring

### 11. What They Do Better Than roxabi_boilerplate

- Specific areas where this repo clearly outperforms roxabi
- Concrete examples with file paths or links
- Actionable suggestions for what roxabi should adopt

### 12. What They Do Better Than 2ndBrain

- Specific areas where this repo outperforms 2ndBrain
- Concrete examples with file paths or links
- Cross-reference with roxabi comparison (are the strengths the same?)

### 13. Key Takeaways

- Top 3-5 actionable items to bring back to roxabi
- Priority ranking (must-have vs nice-to-have)
- Links to potential GitHub issues to create
